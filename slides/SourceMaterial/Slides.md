- Least resource-intensive usable version of an object.

# Audio training sequence—traditional

1. practice—music, storytelling (theater), tabletop roleplay
2. recording (e.g., soundwalking, performance documentation, actual play podcast)
3. editing (e.g., Audacity)
4. routing (e.g., Reaper)
5. control and cueing (e.g., Ableton, QLab, Traktor)
    1. MIDI manipulation
6. installation audio, visual/flowchart programming (Max)
    1. bespoke software engineering (e.g., Web Audio SDK)
7. gaming middleware (FMOD, Wwise)
    1. visual flowchart programming (e.g., Unreal MetaSounds)
        1. bespoke software engineering (e.g., plugin adaptation for Wwise shipping)

## Ergatogeny repeats phylogeny

Ergatogeny: professional ontology, the emergence of a skilled worker (_ergat_-):

A student (who is always already a worker) develops skills according to a sequence recapitulating the history of their craft.

## Problems

- Perception that timelines are easier than cyclic playtime: Fixed-medium timeline mindset is useful for replicable data (annotation, reference) but wrong for user experience. Linear time is also ethnocentric.
- Middleware nudges students: specialize in audio, underpaid; separates students from interactive logic.
- GUI impairs software architecture awareness, blocks access to transferable & generalizable tools.
- Artificially places instructors ahead of students in a linear development process.
